*आराम से बैठें, एक प्रॉम्प्ट लिखें, और बाकी काम 'कॉच' (ComfyUI) को करने दें।*

अपने एडिटर को छोड़े बिना ComfyUI के साथ इमेज और वीडियो जेनरेट करें।
एक प्रीसेट चुनें, एक प्रॉम्प्ट लिखें, और स्टेटस बार पर नज़र रखें, जबकि CodeComfy वर्कफ़्लो सबमिशन, पोलिंग, फ्रेम डाउनलोड और FFmpeg असेंबली को संभालता है।

> **विंडोज के लिए, लेकिन अन्य प्लेटफॉर्म पर भी काम करने योग्य।** विंडोज 10/11 पर पूरी तरह से परीक्षण किया गया।
> macOS और Linux पर भी काम करने की उम्मीद है - [ज्ञात सीमाओं](#known-limitations) को देखें।
> पुल रिक्वेस्ट (PR) का स्वागत है।

## आवश्यकताएं

| निर्भरता | आवश्यक | टिप्पणियाँ |
|------------|----------|-------|
| **ComfyUI** | हाँ | स्थानीय रूप से (`http://127.0.0.1:8188`) या किसी दूरस्थ मशीन पर चल रहा है। CodeComfy इसके HTTP API के साथ संवाद करता है। |
| **FFmpeg**  | वीडियो के लिए | यह आपके सिस्टम के PATH में होना चाहिए *या* `codecomfy.ffmpegPath` के माध्यम से कॉन्फ़िगर किया जाना चाहिए। [FFmpeg डाउनलोड करें](https://ffmpeg.org/download.html)। |
| **NextGallery** | वैकल्पिक | कम्पैनियन गैलरी व्यूअर। यह स्वयं इमेज जेनरेशन के लिए आवश्यक नहीं है। |

## इंस्टॉलेशन

CodeComfy अभी तक VS Code मार्केटप्लेस पर उपलब्ध नहीं है।
`.vsix` फ़ाइल से इंस्टॉल करें:

1. नवीनतम `.vsix` फ़ाइल को यहां से डाउनलोड करें:
[रिलीज़](https://github.com/mcp-tool-shop-org/codecomfy-vscode/releases)।
2. VS Code में: **एक्सटेंशन** साइडबार → `···` मेनू → **VSIX से इंस्टॉल करें...**
3. जब संकेत मिले तो विंडो को रीलोड करें।

### सेटिंग्स

**सेटिंग्स → एक्सटेंशन → CodeComfy** खोलें या `settings.json` में जोड़ें:

```json
{
  "codecomfy.comfyuiUrl": "http://127.0.0.1:8188",
  "codecomfy.ffmpegPath": "",
  "codecomfy.autoOpenGalleryOnComplete": true,
  "codecomfy.nextGalleryPath": "",
  "codecomfy.defaultNegativePrompt": ""
}
```

| सेटिंग | विवरण | डिफ़ॉल्ट |
|---------|-------------|---------|
| `codecomfy.comfyuiUrl` | ComfyUI सर्वर URL | `http://127.0.0.1:8188` |
| `codecomfy.ffmpegPath` | FFmpeg निष्पादन योग्य का पूर्ण पथ (PATH लुकअप के लिए खाली छोड़ दें) | `""` |
| `codecomfy.autoOpenGalleryOnComplete` | जेनरेशन पूरा होने के बाद NextGallery खोलें | `true` |
| `codecomfy.nextGalleryPath` | NextGallery.exe का पूर्ण पथ | ऑटो-डिटेक्ट |
| `codecomfy.defaultNegativePrompt` | जेनरेशन के दौरान डिफ़ॉल्ट नकारात्मक प्रॉम्प्ट पहले से भरा हुआ | `""` |

## क्विकस्टार्ट

1. **ComfyUI शुरू करें** — सुनिश्चित करें कि यह चल रहा है और एक्सेस किया जा सकता है।
2. **एक कमांड चुनें** — कमांड पैलेट (`Ctrl+Shift+P`) खोलें और चुनें:
- `CodeComfy: इमेज जेनरेट करें (HQ)` — एक सिंगल इमेज
- `CodeComfy: वीडियो जेनरेट करें (HQ)` — एक छोटा वीडियो (2–8 सेकंड)
3. एक प्रॉम्प्ट दर्ज करें, वैकल्पिक रूप से एक **नकारात्मक प्रॉम्प्ट** (उन चीज़ों से बचें जिन्हें आप नहीं चाहते हैं), और एक **सीड**, फिर स्टेटस बार पर नज़र रखें।

<!-- स्क्रीनशॉट: वास्तविक PNG छवियों से बदलें - assets/SCREENSHOTS.md देखें -->

**स्टेटस बार** वास्तविक समय में प्रगति दिखाता है (कतारबद्ध → जेनरेशन → पूर्ण)।

संरचित लॉग **CodeComfy** आउटपुट चैनल में दिखाई देते हैं
(`Ctrl+Shift+U`, फिर "CodeComfy" चुनें)।

आउटपुट आपके वर्कस्पेस रूट में `.codecomfy/outputs/` में सहेजे जाते हैं।
रन मेटाडेटा `.codecomfy/runs/` में रहता है।

### रद्द करें

कमांड पैलेट से `CodeComfy: जेनरेशन रद्द करें` चलाएं या जेनरेशन के दौरान स्टेटस बार आइटम पर क्लिक करें।

## जेनरेशन सीमाएं

वीडियो जेनरेशन आकस्मिक रूप से संसाधनों के समाप्त होने से रोकने के लिए सुरक्षा सीमाओं को लागू करता है:

| पैरामीटर | न्यूनतम | अधिकतम |
|-----------|-----|-----|
| अवधि | 1 सेकंड | 15 सेकंड |
| FPS | 1   | 60   |
| कुल फ्रेम (अवधि × FPS) | — | 450 |

यदि आप किसी सीमा तक पहुँच जाते हैं, तो अवधि को कम करें या कम फ्रेम दर वाले प्रीसेट का चयन करें।

## समस्या निवारण

### `[नेटवर्क]` — ComfyUI सर्वर तक नहीं पहुंच पा रहा है।

- क्या ComfyUI चल रहा है? जांच करने के लिए ब्राउज़र में `http://127.0.0.1:8188/system_stats` पर जाएं।
- यदि ComfyUI किसी अन्य पोर्ट या होस्ट पर चल रहा है, तो `codecomfy.comfyuiUrl` को अपडेट करें।
- क्या फ़ायरवॉल या प्रॉक्सी कनेक्शन को ब्लॉक कर रहा है? `curl http://127.0.0.1:8188/system_stats` आज़माएं।

### `[सर्वर]` — ComfyUI ने एक त्रुटि लौटाई।

- स्टैक ट्रेस के लिए ComfyUI टर्मिनल/कंसोल की जांच करें।
- सामान्य कारण: गुम मॉडल चेकपॉइंट या कस्टम नोड।
- सुनिश्चित करें कि आपके ComfyUI में उस प्रीसेट वर्कफ़्लो के लिए आवश्यक नोड मौजूद हैं।

### `[एपीआई]` — प्रतिक्रिया आकार में त्रुटि।

- आपका ComfyUI संस्करण बंडल किए गए प्रीसेट के लिए बहुत पुराना या बहुत नया हो सकता है।
- एक रिवर्स प्रॉक्सी या सीडीएन जेएसओएन प्रतिक्रियाओं को विकृत कर सकता है।
- प्रतिक्रिया के आकार की जांच करने के लिए `/prompt` और `/history` को सीधे आज़माएं।

### `[इनपुट/आउटपुट]` — फ़ाइल अनुमति या डिस्क संबंधी समस्या।

- सुनिश्चित करें कि आपका कार्यक्षेत्र फ़ोल्डर लिखने योग्य है।
- उपलब्ध डिस्क स्थान की जांच करें - वीडियो के लिए फ्रेम डाउनलोड बड़े हो सकते हैं।
- विंडोज पर, सर्वोत्तम प्रदर्शन के लिए नेटवर्क ड्राइव पर कार्यक्षेत्रों का उपयोग करने से बचें।

### FFmpeg नहीं मिला।

- FFmpeg स्थापित करें और सुनिश्चित करें कि `ffmpeg.exe` आपके सिस्टम PATH में है।
- या `codecomfy.ffmpegPath` को पूर्ण पूर्ण पथ पर सेट करें (उदाहरण के लिए, `C:\ffmpeg\bin\ffmpeg.exe`)।
- सापेक्ष पथ और बिना नाम (PATH में मौजूद `ffmpeg` के अलावा) सुरक्षा कारणों से अस्वीकार कर दिए जाते हैं।

### "जेनरेशन पहले से ही चल रहा है"

एक ही समय में केवल एक जेनरेशन चल सकता है।
वर्तमान जेनरेशन को रद्द करें (`CodeComfy: जेनरेशन रद्द करें`) या इसे पूरा होने तक प्रतीक्षा करें।
लगातार कार्यों के बीच 2 सेकंड का कूलडाउन होता है।

### सीड/प्रॉम्प्ट सत्यापन।

- सीड पूरे संख्या होने चाहिए जो 0 और 2,147,483,647 के बीच हों।
- प्रॉम्प्ट खाली नहीं होना चाहिए और अधिकतम 8,000 वर्णों तक होना चाहिए।

## सुरक्षा और डेटा दायरा।

- **नेटवर्क:** केवल उपयोगकर्ता द्वारा कॉन्फ़िगर किए गए ComfyUI URL (डिफ़ॉल्ट `127.0.0.1:8188`) से कनेक्ट होता है - कोई अन्य आउटबाउंड अनुरोध नहीं।
- **फ़ाइलें:** आउटपुट `.codecomfy/outputs/` और `.codecomfy/runs/` में कार्यक्षेत्र में सहेजे जाते हैं - कार्यक्षेत्र के बाहर की कोई भी फ़ाइल नहीं बदली जाती है।
- **FFmpeg:** सभी स्पॉन से `shell: true` हटा दिया गया है; पथ पूर्ण, मौजूद और निष्पादन योग्य होना चाहिए।
- कोई भी टेलीमेट्री एकत्र या भेजा नहीं जाता है - पूर्ण नीति के लिए [सुरक्षा.md](सुरक्षा.md) देखें।

## ज्ञात सीमाएँ।

| क्षेत्र। | स्थिति। |
|------|--------|
| **Windows** | पूरी तरह से परीक्षण किया गया (विंडोज 10/11)। प्राथमिक प्लेटफ़ॉर्म। |
| **macOS** | छवि और वीडियो जेनरेशन के लिए काम करने की उम्मीद है। NextGallery अभी तक उपलब्ध नहीं हो सकता है। |
| **Linux** | छवि और वीडियो जेनरेशन के लिए काम करने की उम्मीद है। NextGallery अभी तक उपलब्ध नहीं हो सकता है। |
| **Remote / WSL** | ComfyUI URL VS Code चलाने वाले होस्ट से पहुंच योग्य होना चाहिए। |

मुख्य कार्यक्षमता (प्रॉम्प्ट → ComfyUI → डाउनलोड → FFmpeg असेंबली) है
प्लेटफ़ॉर्म-स्वतंत्र। विंडोज-विशिष्ट सुविधा NextGallery का स्वचालित पता लगाना है, जो अन्य प्लेटफ़ॉर्म पर "सेटिंग में पथ सेट करें" प्रॉम्प्ट पर वापस आ जाता है।

यदि आपको किसी प्लेटफ़ॉर्म-विशिष्ट समस्या का सामना करना पड़ता है, तो कृपया
[एक समस्या खोलें](https://github.com/mcp-tool-shop-org/codecomfy-vscode/issues)
अपने OS, VS Code संस्करण और ComfyUI संस्करण के साथ।

## यह कैसे काम करता है।

```
Command Palette
   │
   ▼
extension.ts  ─── validates inputs, creates JobRouter
   │
   ▼
JobRouter     ─── creates run folder, tracks lifecycle
   │
   ▼
ComfyServerEngine ─── POST /prompt → poll /history → stream /view
   │
   ▼
FFmpeg        ─── (video only) assemble frames → MP4
   │
   ▼
.codecomfy/outputs/index.json  ─── atomic index update
```
